# Infrastructure — WorkspaceContext & VCS — File Index
> Scope: `Infrastructure/{WorkspaceContext,VCS}`. 69 files.

## Subsystem role
`WorkspaceContext` is the context-engineering core: an actor-isolated file catalog (`WorkspaceFileContextStore`) that ingests filesystem deltas in order, indexes files/folders per root, resolves user-supplied paths (alias-aware, fuzzy, case-folded) into concrete records, and powers fast path search. On top of that catalog it manages what gets selected, sliced (line ranges with rebase/anchoring), token-counted, and projected for agent-session worktree bindings. `VCS` is the git/jujutsu substrate beneath the app's diff, status, branch, and worktree features: a backend abstraction (`VCSBackend`) with Git and Jujutsu implementations dispatched by `VCSService`, a `GitStatusActor` that streams repo status snapshots, and a `GitDiff` engine + snapshot store/publisher that compute, persist, and version unified-diff artifacts (including app-managed worktree creation, include-copying, and merge previews). Together: WorkspaceContext decides which files/slices/tokens enter the context window; VCS supplies the diff/status/branch data those flows depend on.
