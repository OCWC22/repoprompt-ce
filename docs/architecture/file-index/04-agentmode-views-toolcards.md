# Agent Mode — Tool Cards & Transcript Views — File Index
> Scope: `Features/AgentMode/Views/{ToolCards,Transcript}`. 37 files.

## Subsystem role
This subsystem is the SwiftUI rendering layer for agent tool activity in the Agent Mode transcript: it turns each `AgentChatItem` (a tool call or its result) into a card, and orchestrates how the transcript scroll view follows the live bottom. The central dispatch is `ToolCardRouter`, an enum whose `callView`/`resultView` factory methods normalize the tool name (`normalizedToolCardName`), match it against `knownResultTools`, and return a per-tool `AnyView` (bash/edit/git/read-search/prompt/context-builder/worktree-merge/etc.), falling back to `UnknownToolResultCard`. Cards share a chrome (`ToolCardContainer`), status model (`ToolCardStatus` + `ToolResultStatusResolver`), accent palette (`ToolCardAccentResolver` over `ClusterToolCategory` families), and an AppKit-backed unified-diff renderer. The Transcript/ folder is the scroll engine: a non-`@Published` `AgentTranscriptScrollEngine` plus pure policy enums (auto-follow rearm, progress/intent resolution, capability/detach decisions) that decide when to stay pinned to the live bottom vs. detach for manual history scrolling.
