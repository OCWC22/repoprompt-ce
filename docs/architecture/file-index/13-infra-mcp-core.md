# Infrastructure — MCP Core Runtime — File Index
> Scope: `Infrastructure/MCP` (flat root files only). 43 files.

## Subsystem role
This is the heart of RepoPrompt CE's bundled MCP server: the long-lived process that exposes RepoPrompt's tools, prompts, and oracle/context-builder workflows to external coding agents (Claude Code, Codex, Cursor, Gemini CLI). It defines the `Service`/`Tool` protocol model and registry, owns the server lifecycle (`ServerController`, `MCPService`), accepts local clients over UNIX-domain and bootstrap sockets, and routes each tool call to the correct window/workspace context via binding resolvers and a routing waiter. It also handles per-call argument normalization, tool-catalog readiness/availability, execution watchdogs, client identity/approval, external-client error surfacing, and JSON config export for installing the server into third-party editors. The subfolders (`Agent/`, `ApplyEdits/`, `AppShared/`, `Policies/`, `ViewModels/`, `WindowTools/`, `WorkspaceApproval/`) build on top of this flat core and are indexed separately.

## Server lifecycle & transport
- `Infrastructure/MCP/MCPService.swift` — background actor owning the single MCP server instance, keeping all network/file I/O off `@MainActor`; publishes status snapshots and multicasts dashboard updates per window. Key: `actor MCPService`, `Snapshot`, `stateStream`, `subscribeToDashboardUpdates`.
- `Infrastructure/MCP/ServerController.swift` — SwiftUI-facing app-level actor singleton managing server status, the persistent always-allowed client allow-list, client-approval dialog flow, and App Nap suppression while running. Key: `actor ServerController`, `shared`, `defaultAlwaysAllowedClients`, `onApprovalRequest`.
- `Infrastructure/MCP/MCPConnectionManager.swift` — large connection-management hub (~9k lines): per-connection MCP server wiring, run-purpose handling, request routing, tool dispatch, policy/availability hooks, and diagnostics for each attached client. Key: `MCPRunPurpose`, connection actors, routing/policy debug gates.
- `Infrastructure/MCP/UnixSocketMCPTransport.swift` — event-driven UNIX-domain socket transport (DispatchSourceRead, no polling) implementing the MCP transport contract with a receive buffer and ordered terminal/cancellation callbacks. Key: `UnixSocketMCPTransport`, callback gate, framed I/O.
- `Infrastructure/MCP/BootstrapSocketServer.swift` — single app-owned UNIX socket listener that replaces filesystem-based discovery; performs the bootstrap handshake and transfers accepted fds to connection managers. Key: `BootstrapSocketServer`, `BootstrapHandshakeSocket`, socket ownership transfer.
- `Infrastructure/MCP/BootstrapSocketConnectionManager.swift` — actor wrapping one MCP connection received over the bootstrap socket from an already-connected fd (no filesystem folders/meta.json). Key: `actor BootstrapSocketConnectionManager: MCPServerConnection`, `isFilesystemBacked == false`.
- `Infrastructure/MCP/BootstrapSocketOwnership.swift` — retained ownership of one well-known socket pathname via a same-directory lock fd; detects stale/replaced/foreign sockets safely. Key: `BootstrapSocketOwnership`, `FileIdentity`, `PathStatus`, `OwnershipError`.
- `Infrastructure/MCP/MCPBootstrapLease.swift` — unified lease actor serializing the entire MCP "bootstrap window" (gate acquisition, routing-waiter registration, policy install, release-on-routed) across agent-mode, headless, and direct flows. Key: `MCPBootstrapLeaseSpec`, lease actor lifecycle, `restrictedTools`/`additionalTools`.
