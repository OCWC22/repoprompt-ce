# Agent Mode — Screen & Chrome Views — File Index
> Scope: `Features/AgentMode/Views` (excl. ToolCards/, Transcript/). 40 files.

## Subsystem role
This subsystem is the SwiftUI "chrome" of Agent Mode — everything around the chat transcript and tool cards: the top-level `AgentModeView` shell, its `NavigationSplitView` controller, the sessions sidebar with per-session rows, the composer/input bar and attachments strip, message bubbles, the composer status/context/workflow/execution pills, the right-hand runtime sidebar (context, context-builder, oracle, export, run status), the onboarding/permissions/models/recommendation popovers, and the Xcode-style titlebar "New Session" accessory. Views are mostly stateless renderers driven by injected view models (`AgentModeViewModel`, `PromptViewModel`, `AgentRuntimeSidebarViewModel`, UI stores like `AgentStatusPillsUIStore`/`AgentComposerUIStore`) and scale geometry through `FontScaleManager`/`FontScalePreset`. ToolCards/ and Transcript/ are indexed separately.
