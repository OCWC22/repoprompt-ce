# Infrastructure — AI Providers — File Index
> Scope: `Infrastructure/AI/Providers`. 65 files.

## Subsystem role
This subsystem holds every concrete `AIProvider`/`HeadlessAgentProvider` implementation behind RepoPrompt's chat and agent surfaces. Two families coexist. **Cloud chat providers** speak HTTP to hosted model APIs — Anthropic (via SwiftAnthropic) and an OpenAI-compatible core (`OpenAIProvider`) that the OpenAI, Azure, Gemini, Grok, Groq, DeepSeek, Fireworks, Ollama, OpenRouter, Z.AI, and arbitrary custom providers either subclass or wrap; `CustomOpenAIProvider` is a hand-rolled HTTP fallback for non-OpenAI-shaped endpoints. **CLI/agent providers** drive local agent binaries as subprocesses: Claude Code via a native NDJSON SDK process protocol (plus a headless `-p` path), Codex via its app-server JSON-RPC and `codex exec`, and Cursor + OpenCode via the Agent Client Protocol (ACP). `AIProviderFactory` is the single switch from `AIProviderType` to a constructed provider (CLI types skip API-key fetching), and `DisposableProviderPool` mints fresh instances per request with key/config fallback caching. The Claude-compatible variants (standard / GLM-Z.ai / Kimi / custom) route their pure logic through `ClaudeCompatibleProviderRuntimeBridge`, the lone import point for the `RepoPromptClaudeCompatibleProvider` package.
