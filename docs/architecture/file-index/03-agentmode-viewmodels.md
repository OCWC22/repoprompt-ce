# Agent Mode — View Models — File Index
> Scope: `Features/AgentMode/ViewModels`. 39 files.

## Subsystem role
This is the MVVM presentation-logic layer for Agent Mode. Its center is one very large `@MainActor` `AgentModeViewModel` (≈14.5k lines in the core file) that owns all per-tab agent chat state — published run state, pending interaction requests (approvals, permissions, MCP elicitations, apply-edits / worktree-merge reviews), transcript presentation, context usage, and attachments. Because the class is so large it is split into many `AgentModeViewModel+*.swift` extension files, each carrying one feature concern (composer props, context-usage accounting, drafts/attachments, git branch switch, interaction actions, provider bindings, sidebar sessions/auto-archive, worktree merge, tab-session inner type, and the type/protocol declarations). To keep SwiftUI re-renders cheap, the view model fans its state out into focused `*UIStore` projection objects (composer, status pills, runtime metrics, session sidebar, transcript, run-interaction) aggregated under `AgentModeUIFacades`; each store holds an `Equatable` snapshot/props struct and only republishes (bumping a revision counter) when the snapshot actually changes. The remaining files cover settings view models (agent models, provider/sub-agent permissions, permission diagnostics), the onboarding and recommendation wizards, and standalone helpers for worktree indicators, workspace-root rows, and sidebar session building / auto-archive policy.
