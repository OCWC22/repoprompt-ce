## Subsystem role
This layer is the macOS app shell: it owns the SwiftUI `App`/`Scene` entry point, the `NSApplicationDelegate` lifecycle, launch-time configuration, global menu/keyboard commands, deep-link/URL/notification routing, the multi-window manager, and the Sparkle update machinery. The composition root is `WindowState` (built by `WindowStateCompositionFactory`): each SwiftUI window gets one `WindowState` that instantiates and wires every per-window view model and service (prompt, oracle, MCP server, workspace manager, agent mode, etc.). `WindowStatesManager` tracks all live `WindowState` instances, persists/restores window sessions, and arbitrates app-wide concerns (MCP continuity, instance numbering, termination). Everything else here (notifications, font scaling, appearance, security hardening, dock/help/update menus) are app-global services that hang off `AppDelegate`/`RepoPromptApp` or are referenced by the windows.
