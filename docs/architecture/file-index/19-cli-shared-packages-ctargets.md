## Subsystem role
These targets are everything in the repo that is *not* the SwiftUI app proper, yet that the app (or its CLI sibling) links against. `RepoPromptMCP` builds the standalone `repoprompt-mcp` executable: the client-side CLI/proxy that external coding agents (Claude Code, Codex, Cursor, Gemini CLI) spawn to reach a running RepoPrompt app over a UNIX bootstrap socket, plus an interactive REPL and a non-interactive exec/scripting mode for driving the same MCP tools by hand. `RepoPromptShared` holds the small set of wire types and POSIX helpers that must be byte-identical on both sides of that socket (control messages, filesystem/socket identity, timeout policy, the JSON-RPC bridge ledger). `Packages/RepoPromptAgentProviders` is an extracted, Foundation-only SwiftPM package staging the Claude-compatible provider codec/translator/catalog logic behind a documented bridge seam so the app can later split it out. The C targets (`RepoPromptC`, `TreeSitterScannerSupport`, vendored `CSwiftPCRE2`) and the Swift PCRE2 wrapper under `ThirdParty` provide the high-performance native substrate — path/scoring search, gitignore wildmatching, tree-sitter scanner ABI fallback, and the PCRE2 regex engine — surfaced into Swift through `Support/RepoPrompt-Bridging-Header.h`.
